# graph-semantics

Complete the semantics of unlabeled variables on a **given causal graph**.

Setting (agreed 2026-07-02): a causal graph is GIVEN (latent and observed nodes, typed edges), together with
labels (short texts) for a SUBSET of the observed variables. The tasks:

- **Task 1** — complete the semantics of the unlabeled OBSERVED variables.
- **Task 2** — Task 1 + translate the LATENT variables.
- (Task 3, later — extend to LLM-scale settings such as agent communication; not in this repo yet.)

**Core problem: optimize latent (and unlabeled-observed) semantic EMBEDDINGS under the causal constraints
derived from the given graph.** Decoding embeddings to natural language uses an existing method (SpLiCE onto
a fixed WordNet dictionary) and is treated as downstream, not the research core.

## Method (v1)

Constraints are read off the given graph:

- **Edges = generation channels.** Every node with parents must be generated by its parents:
  `gen(i) = Σ_{p ∈ pa(i)} W[i,p] · e_p`, with edge weights `W` estimated from the data on the given support
  (latent scores = first PC of the latent's observed descendants; weights = correlations).
- **Absent connections = independence.** Marginally independent pairs (no directed path, no common ancestor)
  have their embeddings decorrelated.
- Norm regularization keeps embeddings on the encoder's unit sphere (decode stability).

Objective, over the free embeddings (latents `u_j` and unlabeled observed `s_i`):

```
min   Σ_{i labeled}    || gen(i) − a_i ||²          (visible labels must be generated by the graph)
    + Σ_{i unlabeled}  || s_i − gen(i) ||²          (unlabeled observed = generated from parents)
    + Σ_{child latent} || u_c − gen(c) ||²          (latent hierarchy consistency)
    + λ_zero Σ_{(p,q) independent} cos²(e_p, e_q)   (independence decorrelation)
    + λ_norm Σ (||e|| − 1)²
```

Labeled embeddings `a_i` come from a frozen sentence encoder (all-MiniLM-L6-v2) and are never optimized.

## Evaluation

K-fold masking over the observed labels (labels hidden, data kept). Metrics, all arms on the same ruler:

- **judge-ACC** — LLM judge (default `gpt-5.5`, calibrated dominant-meaning prompt) on the SpLiCE-decoded words;
- **matching-ACC** — one-to-one assignment of a fold's predictions to its held-out labels (identity, LLM-free);
- **exact top-1** — strict reference.

Baselines: uniform (no structure), raw correlation (no graph). Full per-item records are saved as JSON.

## Testbeds (real data; graphs from released files / study design)

| dataset | graph | observed | latents |
|---|---|---|---|
| TLVD Multitasking | TLVD's released RLCD `.dot` (hierarchical) | 9 task scores | 4 (their GT descriptions ship in their description file) |
| Himi | study-design bipartite (task → construct) | 17 task scores | 6 constructs |
| Big Five | design bipartite (item → factor) | 50 items | 5 factors |

Data files are NOT committed. Set `GRAPHSEM_DATA` (default `../data` relative to this repo) so that:
- `data/TLVD/Final_Multitasking_Data.sav` (OSF: project tn6hp)
- `data/BIG5/{codebook.txt,data.csv}` (openpsychometrics.org IPIP-FFM)
- the WordNet dictionary npz via `GRAPHSEM_DICT` (words + 384-d MiniLM embeddings)
Run `bash fetch_tlvd.sh` once to cache TLVD's released graph/description files into `data_cache/`.

## Run

```
export OPENAI_API_KEY=...          # judge only; optional (falls back to geometric metrics)
python run_task1.py                # observed-label completion, all testbeds
python run_task2.py                # + latent translation
```

Env knobs: `DATASET=tlvd|himi|bigfive|all`, `FOLDS`, `LAM_ZERO`, `LAM_NORM`, `STEPS`, `DEVICE=cpu|auto|cuda`,
`EDGE_WEIGHT_MODE=signed|abs|positive`, `NORMALIZE_GEN=0|1`, `LAM_OBS_PRIOR`, `OBS_PRIOR_SCOPE=siblings|all`,
`JUDGE_MODEL`, `RECORDS_OUT`.

## Honest notes

- The judge and the matching metric measure different things (semantic correctness vs individual identity);
  we report both.
- On TLVD's graph the latent ground truth uses the 4 construct descriptions found in TLVD's own released
  description file; the latent→description mapping is stated in `datasets.py` and is a derivation, not a
  released artifact.
- λ's are fixed defaults, not tuned per dataset.
