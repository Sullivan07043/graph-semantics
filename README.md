# graph-semantics

Complete the semantics of unlabeled variables on a **given causal graph**.

Setting (agreed 2026-07-02): a causal graph is GIVEN (latent and observed nodes, typed edges), together with
labels (short texts) for a SUBSET of the observed variables. The tasks:

- **Task 1** — complete the semantics of the unlabeled OBSERVED variables.
- **Task 2** — Task 1 + translate the LATENT variables.
- (Task 3, later — extend to LLM-scale settings such as agent communication; not in this repo yet.)

**Core problem: optimize latent (and unlabeled-observed) semantic EMBEDDINGS under the causal constraints
derived from the given graph.** Decoding embeddings to natural language uses an existing method (SpLiCE onto
a fixed dictionary) and is treated as downstream, not the research core.

## Line A — structured optimization (training-free core)

Constraints read off the given graph; variables are the embeddings themselves plus (optionally)
embedding-space edge magnitudes and per-node residual vectors:

```
min   Σ_{i labeled}    || gen(i) − a_i − r_i ||²        visible labels generated by the graph
    + Σ_{i unlabeled}  || s_i − gen(i) − r_i ||²        unlabeled = generation + own residual
    + Σ_{child latent} || u_c − gen(c) − r_c ||²        latent hierarchy consistency
    + μ Σ ||r_i||²                                      residuals stay small
    + λ_res Σ_{(i,k)} ( cos(r_i,r_k) − ρ̂_ik )²          residual directions ≈ data partial correlations
    + λ_zero Σ_{(p,q) independent} cos²(e_p, e_q)       absent connections = decorrelation
    + λ_norm Σ (||e|| − 1)²                             stay on the encoder sphere (decode stability)
```

with `gen(i) = Σ_{p ∈ pa(i)} W̃[i,p] · e_p`. Edge support and sign are locked to the given graph + data;
with `free_w` the magnitudes W̃ are optimized too (data-space correlations need not equal embedding-space
coefficients). Solver: deterministic ALS (closed-form least squares for embeddings ↔ sign-constrained
NNLS for magnitudes), then Adam refinement for the non-quadratic terms. Labeled embeddings a_i come from a
frozen sentence encoder and are never optimized.

## Line B — trained completion operator (gnn.py)

A typed message-passing network g_θ over the given graph (9 relations: 4 edge types × 2 directions +
data-correlation top-k neighbours), trained on the DEV pool only via masked-label reconstruction:
hide a random subset of observed labels, reconstruct their frozen-encoder embeddings (cosine loss +
generation-consistency prior). θ is shared across datasets; held-out datasets are evaluated ZERO-SHOT.
The frozen encoder is never fine-tuned (inputs and targets live in its fixed space — no semantic drift).

## Evaluation protocol (LODO, fixed 2026-07-14)

- **Dev pool** (all global choices fitted here, and ONLY here): tlvd, himi, bigfive, hs, rse, mach, gcbs.
- **Held-out** (never touched during development; frozen method, run once): hexaco (3-layer
  factor→facet→item), sixteenpf, riasec, hsq, kims, sd3.
- Within each dataset: 5-fold masking over observed labels (labels hidden, data kept).
- Metrics: judge-ACC (gpt-5.5, calibrated dominant-meaning prompt) on SpLiCE-decoded words; matching-ACC
  (Hungarian, identity, LLM-free); MRR of the true label among all dataset labels (encoder-invariant);
  exact top-1. Dev iteration uses the geometric metrics; the judge is spent on the final runs.
- Baselines: uniform, raw correlation (no graph), LLM-naming (Task 2).

## Global choices (fitted on dev, then frozen)

- **Encoder: intfloat/e5-large-v2** (symmetric "query: " prefix). Bake-off on dev, encoder-invariant
  metrics (matching-ACC): e5-large 0.737 > gte-large 0.704 > MiniLM 0.625. `GRAPHSEM_ENCODER` env.
- **Dictionary: WordNet concept bank ∪ ConceptNet Numberbatch English phrases** (~521k terms, 1-3 words),
  embedded with the frozen encoder (`build_dictionary.py`); SpLiCE decode uses cosine top-2000 screening.
  One dictionary for all datasets; α picked globally (ℓ0 ≈ 8), never per dataset.
- λ's and the (free_w, μ, λ_res) configuration: selected on the dev-pool ablation (`ablate.py`).

## Run

```
export OPENAI_API_KEY=...                 # judge only
python run_task1.py                       # DATASET=dev|heldout|all|<name>
python run_task2.py
python bakeoff.py                         # encoder bake-off (dev, geometric)
python ablate.py                          # line-A ablations (dev, geometric)
python gnn.py train && python gnn.py eval dev|heldout
python build_dictionary.py                # rebuild decode dictionary for the current encoder
```

Env knobs: `DATASET`, `FOLDS`, `STEPS`, `LAM_ZERO`, `LAM_NORM`, `FREE_W`, `RESIDUAL` (μ), `LAM_RES`,
`GRAPHSEM_ENCODER`, `GRAPHSEM_DICT`, `JUDGE_MODEL`, `RECORDS_OUT`.

## Honest notes

- The judge and the matching metric measure different things (semantic correctness vs individual
  identity); we report both. Negative-loading items decode via the antonym direction ("low X" prefix)
  when the nonnegative decode is empty.
- On TLVD's graph the latent ground truth uses TLVD's own released construct descriptions; the
  latent→description mapping is a stated derivation. KIMS item 21 is keyed to Observe per the published
  instrument (the site's scoring code omits it). GCBS item texts come from the published scale, not the
  data dump.
- Raw cosine is NOT comparable across encoders; encoder selection used matching-ACC and MRR only.
